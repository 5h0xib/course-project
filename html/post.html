Everything you should know about 'module' & 'require' in node.js

Srishti Gupta

Node.js treats each JavaScript file as a separate module. For instance, if you have a file containing some code and this file is named xyz.js, then this file is treated as a module in Node, and you can say that you’ve created a single module named xyz. Let’s take an example to understand this better. You have a file named circle.js which consists of the logic for calculating the area & circumference of a circle of a given radius. You can call circle.js a module named circle. You might be wondering why is there need to have multiple modules? You could have just written all the code in single module. Well, it is very important to write modular code. By modular, I mean to say that your code should be independent and should be loosely coupled… Imagine that there’s a large application and you have all your code written in just one place, just one file… Too messy right?
